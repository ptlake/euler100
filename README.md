# euler100

These files contain my solutions to the first 100 problems of [ProjectEuler](https://projecteuler.net/) as coding examples in the form a Jupyter Notebook.  This format allows me to explain my thought process for each piece of code along with the code itself.

My degree is in Chemistry, and it is unfortunate that in itself does not fully do justice in showing that I also can do Computer Science and Mathematics.  I feel that hosting a set of coding examples on GitHub is arbitrary, boring to work on, and difficult for someone else to have interest in.  The coding exercise that I have enjoyed doing have all been on platforms that typically scorn sharing solutions, which makes too much sense.  When I saw that ProjectEuler promotes sharing solutions to the first 100 problems, I decided it would be good to work on using it as coding examples curated by an outside source.

I have found the problems on ProjectEuler satisfying in that it exemplifies both the need to have a good math background to answer numerical problems, but also emphasize the importance of computer programming to be able to off-load the drudgery of actually manipulating the numbers to a computer.  It also brings to light the challenge of finding the appropriate algorithms and data structures that allow for efficient calculation.

I have worked on a few hundred problems on ProjectEuler and one thing that is unfortunate is how the first 100 problems are comparatively easy.  There is not as much need to develop an answer in a specific way for these problems, usually the first thing that comes to mind is more than sufficient to answer the question.  For this reason, most of the problems I present that knee-jerk first thought that can be coded instantaneously to get the answer no matter how inefficient it is computationaly.  Then I work out a more efficient code that is more involved but can extend the limits of the calculation way past what is being asked for in the problem.

Hopefully it is self-explanatory that the purpose of these files are not for people to use to solve ProjectEuler problems.  For anyone that comes across this work who is trying the problems for the first time, I advise to solve the problem by yourself first.  Then you can turn with morbid curiosity to see how other people have done it, and then, perhaps, gloat that you have done it better.
